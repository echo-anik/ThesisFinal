EDGE PERCENTAGE ANALYSIS
========================

Optimal Edge Training Data:
- 5% (42k samples): Works well, lower central load
- 10% (84k samples): Better edge quality, minimal overhead
- 15% (126k samples): Diminishing returns visible
- 20%+ (168k+ samples): Marginal improvement

Recommendation: Use 10% for edge training
- Balances edge model quality with training time
- Keeps escalation minimal (1.8-2% for Config_2)
- Total training time: ~5 minutes

HOW TO USE THIS CONFIGURATION
=============================

Step 1: Train Models (Server)
  python run_final_hai_experiments.py
  - Generates trained models
  - Saves to pickle files
  - Runtime: 30-40 minutes

Step 2: Deploy to Edge (Raspberry Pi)
  scp edge_model.pkl pi@192.168.1.100:/home/pi/ids/
  - Copy edge model to Pi
  - File size: ~6MB (fast transfer)

Step 3: Run Edge Daemon
  python edge_ids_daemon.py
  - Runs on Pi 24/7
  - Monitors sensor input
  - Makes predictions
  - Escalates uncertain cases

Step 4: Handle Escalations (Central Server)
  python central_api_server.py
  - Receives uncertain cases
  - Runs heavy computation
  - Returns final decision
  - Logs all events

Step 5: Monitor & Maintain
  - Check daily false alarm logs
  - Monitor Pi CPU/memory
  - Verify network connectivity
  - Quarterly model retraining
